🚖 Call Taxi Booking Application

This project is a console-based Call Taxi Booking Application developed using Object-Oriented Programming (OOP) concepts. The system simulates a simplified taxi booking service where customers can book taxis that operate between predefined pickup and drop locations on a linear route.

The application manages multiple taxis, assigns them to customer requests based on availability and proximity, calculates trip fares, and maintains booking history along with each taxi’s total earnings.

The route consists of six points A, B, C, D, E, and F, where the distance between each point is 15 km, and the travel time between two consecutive points is 60 minutes. Initially, all taxis are stationed at point A. When a customer requests a ride, the system allocates a taxi according to specific booking rules such as taxi availability, distance from the pickup location, and the taxi’s total earnings.

The fare calculation follows a simple pricing model where the minimum charge is ₹100 for the first 5 km, and ₹10 per km is charged for the remaining distance. The system also computes the drop time based on the travel distance between the pickup and drop locations.

The application includes functionality to:

Book taxis for customers based on pickup location, drop location, and pickup time.

Automatically allocate the most suitable taxi according to defined rules.

Calculate trip fare and travel time.

Store and manage booking records.

Display detailed taxi reports including trip history and total earnings.
This project demonstrates important software design concepts such as Low Level Design (LLD), object-oriented modeling, class relationships, and logical problem solving, making it a good practice project for learning system design and preparing for technical interviews.
